setInterval() / setTimeout()

    - 두 함수 모두 Job Scheduler(스케쥴러(대기 등))를 지원한다.
    - 일반적 차이점 : 
      setInterval() : 지정된 주기로 특정 코드를 계속해서 실행한다. clearInterval()을 만나면 수행 종료.  // 메모리 누수 발생할 수 있음
      setTimeout() : 지정된 시간(1/1000초(m/s))이 지나면 특정 코드를 1회 수행한다. 단, 자신을 호출하면 반복실행가능. clearTimeout()  // 정해진 시간만큼 수행
    - 메모리적인 차원 : 두 함수 모두 가비지 컬렉터의 대상이 아니다. 따라서 메모리에 독립적으로 유지된다.(해제하지 않으면 메모리에 계속 남아있음)
